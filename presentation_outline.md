# 人类工效学：智能交互时代的人本设计

> **课程**：智能交互技术（研究生）  
> **建议时长**：18–22 分钟 · 约 12–15 张幻灯片  
> **结构**：开场 + 5 个正式章节  
> **两条主线**：交互 × 智能

---

## 第〇部分：开场——一张"反面教材"（~1 min / 1 张 slide）

- 第一页**故意设计成糟糕的幻灯片**：低对比度、字体混乱、信息过载、色彩刺眼
- 第二页**瞬间切换**为规范的、舒适的设计
- 问观众：**"哪一页让你更难受？为什么？"**
- 引出核心问题：**设计不以人为本，是有代价的**
- 过渡语：*"这就是今天要聊的——人类工效学"*

## 第一部分：人类工效学是什么（~3 min / 2–3 张 slides）

### 1.1 学科命名与定义

- **学科名称的多元性**（体现交叉学科特征）：
  - 美国：Human Factors Engineering（人的因素工程学）/ Human Engineering（人类工程学）
  - 西欧：Ergonomics（人类工效学）
  - 日本：人间工学
  - 中国：人机工程学、人体工程学、工程心理学、人类工效学等
  - 目前国内外尚无统一名称，使用"人因工程学"和"人类工效学"较多

- **IEA 定义（2000年）**：
  > 研究人与系统中其他因素之间的相互作用，以及应用相关理论、原理、数据和方法来设计以达到优化人和系统效能的学科。

- **核心理念**（一句话锚定）：
  > *"系统如何适应人，而不是人去适应系统"*

- **人—机—环境系统**：由共处于同一时间和空间的人与其所操纵的机器以及所处的周围环境构成的系统
  - 目标：使人能**高效、安全、健康、舒适**地工作和生活

### 1.2 三大子领域

| 子领域 | 研究内容 | 关键词 |
|--------|----------|--------|
| **物理工效学** | 姿势、力、环境、人体测量、生物力学 | 身体适配 |
| **认知工效学** | 注意力、记忆、决策、信息加工、心智模型 | 脑力适配 |
| **组织工效学** | 工作流、团队协作、组织管理、沟通方式 | 系统适配 |

### 1.3 学科边界

- **不是** UI 设计，**不是** 心理学，而是横跨**工程、医学、行为科学**的交叉学科
- 构成体系：
  - **人体科学**：生理学、心理学、人体测量学、解剖学、生物力学
  - **技术科学**：工业设计、工程设计、系统工程、安全工程、机械工程
  - **环境科学**：环境保护学、环境医学、环境心理学、环境测量学

## 第二部分：为什么要研究它（~3 min / 2 张 slides）

### 2.1 代价是真实的（数据支撑）

- 美国每年因**重复性劳损**造成的工伤占工伤总量约 **1/3**
- 航空事故中**人为因素**占 **70%+**
- 人因工程萌芽期（20世纪初）的教训：以机器为中心 → 人去适应机器 → 大量伤害人身心的问题

### 2.2 历史演进中的启示

| 时期 | 重点 | 关键事件 |
|------|------|----------|
| **萌芽期**（20世纪初） | 提高生产效率 | 泰勒铁铲实验、吉尔布雷斯动作研究、闵斯托伯格《心理学与工业效率》 |
| **兴起期**（一战→二战前） | 减轻疲劳、人适应机器 | **霍桑实验**（8年）：工作效率不仅受物理/生理因素影响，组织因素和人际关系同样重要 |
| **成长期**（二战→60年代） | 人机界面匹配 | 从"人适应机器"转向"机器适应人"；1960年成立IEA |
| **发展期**（60年代至今） | 人-机-环境最佳匹配 | 应用范围扩展至各行各业；在高技术领域发挥特殊作用 |

### 2.3 工业界与学术价值

- **工业界**：Apple、Google、Meta 等公司均设有专门的 Human Factors 团队，早于产品研发介入
- **学术价值**：提供**可量化的评估框架**：
  - 任务完成时间
  - 错误率
  - 认知负荷量表（**NASA-TLX**）
  - 系统可用性量表（**SUS**）
  - 弥补了纯定性用户研究的不足

## 第三部分："人的因素"——交互的生理与心理基础（~4 min / 3 张 slides）

### 3.1 神经系统——交互的硬件基础

- **中枢神经系统**（脑+脊髓）：人体的主导系统
  - 大脑皮层：2-5mm 厚的灰质细胞层，两半球协调活动
  - 信息传导通路：感觉 → 脊髓 → 脑 → 运动指令
- **神经元与突触**：
  - 神经元 = 细胞体 + 突起（树突、轴突）
  - 突触传递：神经冲动 → 递质释放 → 突触后膜去极化 → 继续传导
- **与交互设计的关联**：反应时间、信息处理通道容量的生理基础

### 3.2 感觉系统——人机交互的输入通道

| 感觉通道 | 核心特性 | 交互设计启示 |
|----------|----------|-------------|
| **视觉** | 短时间获取大量信息；分辨率高；但易疲劳、易产生错觉 | 界面设计首选通道，注意对比度、色彩恒常性、避免错觉 |
| **听觉** | 声压超过听阈才能感知；不同频率听阈不同 | 语音交互需考虑噪声环境、频率范围 |
| **触觉/肤觉** | 包括触、压、振、温、痛；感受器在皮肤表层 | 触觉反馈设计（振动、力反馈） |
| **平衡觉** | 前庭器官感受加速和旋转运动 | VR/AR 晕动症的根源 |
| **味觉/嗅觉** | 反应速度慢（味觉需1s），恢复慢 | 多模态交互的辅助通道 |

- **感觉 → 知觉**的提升：知觉具有**整体性、选择性、理解性、恒常性**，也会产生**错觉**
  - 经典知觉规律：接近、相似、封闭、连续（格式塔原理）
  - 这些规律直接指导UI布局和信息架构设计

### 3.3 心理因素——影响交互行为的内在变量

- **性格**：先天+后天，影响用户使用习惯和容错需求
- **能力**：
  - 一般能力（观察力、记忆力、创造力）
  - 操作能力、认知能力、社交能力
  - 影响因素：素质、知识、教育、环境、实践、主观努力
- **动机**：
  - 中等强度动机 → 最高工作效率（**倒U曲线/Yerkes-Dodson定律**）
  - 动机过强反而阻碍表现
- **情绪**：心境（持久微弱）、激情（强烈短暂）、应激（意外适应性反应）
- **意志**：自觉确定目的并调节行为的心理过程

### 3.4 脑力劳动的生理变化

- 脑重仅占体重 **2.5%**，但耗氧量达全身 **10%**
- 安静时脑的氧代谢约为等量肌肉的 **15–20倍**
- 紧张时：心跳加快、血压上升、呼吸加快、脑部充血、四肢血液减少
- **启示**：长时间高认知负荷任务需要合理的休息与界面简化

## 第四部分：人机系统——从理论到智能交互（~4 min / 3 张 slides）

### 4.1 人机系统的基本模型

- **系统组成**：人的子系统 + 机器的子系统 + 人机界面
  - 人：感受刺激 → 大脑信息加工 → 做出反应（**S-O-R**）
  - 机器：控制装置 → 机器运转 → 显示装置（**C-M-D**）
  - 关键：人与机之间的**信息环路**能否持续有效地运行

- **三种分类维度**：

| 分类方式 | 类型 | 说明 |
|----------|------|------|
| 按反馈控制 | 闭环 / 开环 | 输出是否作用于控制 |
| 按自动化程度 | 手工操作 / 半自动 / 全自动 | 人的角色从操作者到监视者 |
| 按结合方式 | 串联 / 并联 / 混合 | 影响系统可靠性 |

### 4.2 人机功能分配——核心设计问题

- **人优于机器的方面**：
  - 感知觉的灵活性（音色分辨、嗅觉、触觉）
  - 多通道信息接收与补偿
  - 高度灵活性和可塑性，能随机应变
  - 归纳推理、经验总结、创新能力
  - 有感情、意识和能动性

- **机器优于人的方面**：
  - 巨大且精确的动力输出
  - 高速信息处理和传递
  - 高精度、高稳定性，无疲劳
  - 同时完成多种操作
  - 在恶劣环境下工作

- **分配原则**：
  - 人承担：程序设计、意外处理、变化频繁的作业、研究决策
  - 机器承担：单调重复、危险作业、高精度/高速操作、恶劣环境

- **关键匹配关系**：
  - 显示器 ↔ 人的感觉通道特性
  - 控制器 ↔ 人体运动反应特性
  - 显示器 ↔ 控制器之间的一致性
  - 环境条件 ↔ 人的生理/心理特性

### 4.3 人机系统设计思想的演变

| 阶段 | 设计思想 | 对应交互范式 |
|------|----------|-------------|
| 第一阶段 | 人适应机器 | CLI（命令行）时代 |
| 第二阶段 | 机器适应人（人机界面设计） | GUI（图形界面）→ 触屏 |
| 第三阶段 | 人与机器相互适应（系统设计） | 语音、手势、空间计算（VR/AR） |

- **每一次交互范式转移都重新定义了工效学的研究问题**：
  - 触屏时代：无触感反馈、"胖手指"效应、单手操作拇指区
  - 语音交互：歧义、噪声环境准确率下降、心智模型建立困难
  - 空间计算（VR/AR）：**"大猩猩手臂"效应**、视觉疲劳、空间晕动症（平衡觉相关）

### 4.4 人机系统评价方法（简要提及）

- **连接分析法**：用"连接"表示人-机关系，计算综合评价值 = 重要程度 × 频率
- **操作顺序图法（OSD）**：分析"信息→决策→动作"的作业顺序，评估反应时间和可靠性
- **校核表法**：针对信息显示、操纵装置、作业空间、环境要素的系统性检查
- **海洛德法（HERALD）**：基于人的最佳视野（水平视线±15°），用劣化值计算有效作业概率
- **工作环境指数法**：密集指数、可视性指数、会话指数、步行指数

## 第五部分：AI 时代的新范式挑战 × 对研究生的启示（~4 min / 2–3 张 slides）

### 5.1 AI 带来的工效学新问题

- **范式转变**：从"静态界面工效学"到"动态、自适应、不可预测系统"的工效学
  - 人无法预知 AI 的输出 → 传统的人机信息环路模型被打破
  
- **可解释性问题**：
  - AI 黑盒决策破坏了工效学中**"系统可见性"（Visibility）原则**
  - 用户无法建立正确的心智模型（对应知觉的"理解性"特性）

- **人机信任校准**：
  - **过度信任**（Automation Bias）→ 人成为橡皮图章
  - **信任不足**（Disuse）→ 系统效率下降
  - 这是当前 Human-AI Interaction 研究的核心议题

- **自适应界面的工效学悖论**：
  - 系统越"聪明"，用户越难预测其行为，反而可能**增加认知负荷**
  - 与知觉的"恒常性"需求冲突——用户期望界面行为稳定可预期

- **案例**：特斯拉全触屏车载系统的工效学争议
  - NHTSA 研究：驾驶时视线转移时间超标
  - 与海洛德法的最佳视野原则（±15°）形成鲜明对照

### 5.2 对 AI 研究生的启示

- **做系统时**：
  - 准确率不是唯一指标——一个 95% 准确但让用户困惑的 AI 系统，**不如** 85% 准确但可预期的系统
  - 人机功能分配仍然适用：AI 擅长的（高速计算、海量数据处理）让 AI 做，人擅长的（归纳推理、意外处理、创新决策）留给人

- **做研究时**：
  - 用工效学评估框架量化用户体验：
    - **NASA-TLX**（认知负荷量表）
    - **SUS**（系统可用性量表）
    - 任务完成时间、错误率
  - 这些是让系统论文更有说服力的有力补充

- **设计智能体（Agent）时**：
  - 保留**"人在回路"（Human-in-the-Loop）**的干预节点
  - 这是工效学视角对自动化系统的核心要求
  - 对应人机系统中的"人监控机器 + 机器监督人"的双向设计

### 5.3 开放问题（留给观众思考）

> 当 AI 系统可以完全替代物理操作，工效学研究的边界在哪里？  
> **认知工效学**是否会成为主战场？
